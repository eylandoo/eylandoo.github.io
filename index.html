مستندات فنی API Technical API Docs

راهنمای کامل و تخصصی برای توسعه‌دهندگان جهت یکپارچه‌سازی پنل مدیریت OpenVPN. تمام ابزارهای لازم برای مدیریت کاربران، سرورها، سیستم فروش و نمایندگی در اختیار شماست.

A complete and technical guide for developers to integrate the OpenVPN Manager Panel. All tools for user management, servers, sales systems, and resellers are at your disposal.

Base URL
https://your-domain.com:PORT
Format
JSON / REST

احراز هویتAuthentication

مدیریت دسترسی و کلیدهای APIAccess control and API Keys

تمام درخواست‌های API باید شامل هدر X-API-KEY باشند. سیستم بر اساس کلید ارائه شده، نقش کاربر (ادمین اصلی یا نماینده) را تشخیص داده و دسترسی‌ها را محدود می‌کند.

All API requests must include the X-API-KEY header. The system automatically detects the role (Main Admin or Sub-Admin) based on the key and restricts permissions accordingly.

ادمین اصلیMain Admin

دسترسی کامل (Root). قابلیت مدیریت نودها، پورت‌ها، نمایندگان و تمام کاربران سیستم. دسترسی به تنظیمات سیستمی و SSL.

Full Root Access. Ability to manage nodes, ports, sub-admins, and all users. Access to system settings and SSL.

نمایندهSub-Admin

دسترسی محدود (Scoped). تنها قابلیت مشاهده و مدیریت کاربران زیرمجموعه خود را دارد. عدم دسترسی به تنظیمات سرور یا نودها.

Scoped Access. Can only view and manage their own sub-users. No access to server settings, nodes, or ports configuration.

Header Example 
X-API-KEY: y0ur_s3cur3_ap1_k3y_h3r3
Content-Type: application/json

POST /api/v1/users
Auth Required

قدرتمندترین متد برای ایجاد کاربر. این متد از دو حالت **تکی** و **گروهی (Bulk)** پشتیبانی می‌کند.
نکته فنی: این درخواست به صورت Thread-Safe اجرا می‌شود و پس از ایجاد کاربر در دیتابیس، عملیات سنگین همگام‌سازی با نودها را مدیریت می‌کند. Powerful endpoint for creating users. Supports both **Single** and **Bulk** creation modes.
Technical Note: This request is Thread-Safe. It handles database creation and heavy node synchronization tasks.

پارامترهای ورودی (JSON Body) Input Parameters (JSON Body)

username
String
Conditional
نام کاربری (حروف انگلیسی و اعداد). اگر bulk_count ارسال نشود، این فیلد اجباری است. Username (alphanumeric). Required if bulk_count is not provided.
bulk_count
Integer
Optional
تعداد کاربر برای ساخت گروهی. در صورت ارسال، نام‌های کاربری به صورت تصادفی تولید می‌شوند و فیلد username نادیده گرفته می‌شود. Count for bulk creation. If sent, random usernames are generated and username field is ignored.
max_clients
Integer
Default: 1
حداکثر تعداد اتصال همزمان. Max concurrent connections.
data_limit
Integer
Optional
مقدار عددی محدودیت حجم. null یا 0 به معنی نامحدود است. Traffic limit value. null or 0 means unlimited.
data_limit_unit
String
Default: 'GB'
واحد حجم. مقادیر مجاز: MB, GB. Traffic unit. Allowed: MB, GB.
activation_type
String
Default: 'fixed_date'
نوع فعال‌سازی: fixed_date (تاریخ ثابت) یا flexible_days (شروع از اولین اتصال). Activation mode: fixed_date or flexible_days (starts on first connect).
expiry_date_str
String (Date)
Conditional
فرمت YYYY-MM-DD. اگر نوع فعال‌سازی fixed_date باشد، این فیلد الزامی است. Format YYYY-MM-DD. Required if activation type is fixed_date.
expiry_days
Integer
Alternative
تعداد روز تا انقضا (جایگزین تاریخ دقیق). تاریخ انقضا = امروز + این تعداد روز. Days until expiry (alternative to exact date). Expiry = Today + this value.
pending_activation_days
Integer
Conditional
تعداد روز اعتبار. اگر نوع فعال‌سازی flexible_days باشد، این فیلد اجباری است. Valid days count. Required if activation type is flexible_days.
nodes
Array[Int]
Optional
لیست ID نودهای مجاز برای کاربر. مثال: [1, 2]. List of allowed node IDs. Example: [1, 2].
notes
String
Optional
یادداشت برای کاربر (نمایش در پنل). User notes (displayed in panel).
sub_admin_id
Integer
Main Admin Only
اختصاص کاربر به یک نماینده خاص. (فقط ادمین اصلی مجاز است). Assign user to a specific sub-admin. (Main Admin only).

PUT /api/v1/users/<username>
Auth Required

ویرایش تنظیمات کاربر. ارسال هر فیلد اختیاری است و فقط مقادیر ارسال شده تغییر می‌کنند.
نکته مهم: اگر کاربر به دلیل اتمام حجم یا انقضا غیرفعال شده باشد، افزایش محدودیت‌ها باعث **فعال‌سازی خودکار** او می‌شود. Update user settings. All fields are optional; only sent values are updated.
Important: Increasing limits for a user disabled by quota/expiry will **automatically re-enable** them.

max_clients
Integer
تعداد اتصال همزمان جدید. New concurrent connections limit.
data_limit
Integer
حجم کل جدید. New total data limit.
data_limit_unit
String
واحد حجم ('MB', 'GB'). Data unit ('MB', 'GB').
expiry_date_str
String
تاریخ انقضا (YYYY-MM-DD). Expiry date (YYYY-MM-DD).
activation_type
String
تغییر نوع فعال‌سازی ('fixed_date', 'flexible_days'). Change activation type ('fixed_date', 'flexible_days').
pending_activation_days
Integer
تعداد روز شناور جدید. New flexible days count.
reset_activation
Boolean
اگر `true` باشد، تاریخ اولین اتصال (First Connection) پاک می‌شود (ریست شناور). If `true`, clears the First Connection date (Resets flexible timer).
nodes
Array[Int]
لیست جدید نودهای مجاز. New list of allowed nodes.
notes
String
یادداشت جدید. New notes.

عملیات‌های کنترلی کاربر User Control Actions

مدیریت وضعیت، ترافیک و امنیت کاربران User status, traffic and security management

POST /api/v1/users/<username>/toggle
تغییر وضعیتToggle Status

تغییر وضعیت آنی کاربر (فعال/غیرفعال). غیرفعال کردن باعث **قطع فوری (Kill)** تمام اتصالات فعال می‌شود. Toggle Active/Inactive state. Disabling triggers an immediate **Kill** of all active sessions. 

{ "success": true, "is_active": false, "message": "User disabled" }
POST /api/v1/users/<username>/reset_traffic
ریست ترافیکReset Traffic

صفر کردن حجم مصرفی. اگر کاربر به دلیل اتمام حجم مسدود شده باشد، به طور خودکار **فعال** می‌شود. Reset usage counters. Automatically **re-enables** user if they were disabled by data limit. 

{ "success": true, "user_re_enabled": true, "message": "Traffic reset." }
POST /api/v1/users/<username>/revoke
منطقه خطرDanger Zone

1. تغییر `token_salt` (لینک اشتراک قبلی بلافاصله باطل می‌شود).
2. صدور مجدد گواهی SSL در پس‌زمینه.
3. قطع تمام اتصالات کاربر. 1. Rotates `token_salt` (Old sub link invalidates instantly).
2. Queues background SSL certificate re-issuance.
3. Kills all active connections. 

{ 
  "success": true, 
  "new_subscription_url": "https://...",
  "message": "Token revoked instantly. Cert generation in background."
}
DELETE /api/v1/users/<username>
Auth Required

حذف دائمیPermanent Deletion

حذف کاربر از دیتابیس، پاک کردن فایل‌ها از سرور اصلی و ارسال فرمان حذف به **تمام نودهای متصل**. این عملیات غیرقابل بازگشت است. Deletes user from DB, removes local files, and syncs deletion command to **all connected nodes**. Irreversible.

پاسخ نمونه Response Example 200 OK 
{
  "success": true,
  "message": "User 'old_client' deleted successfully from all servers."
}

مدیریت نودهاNode Management

کنترل سرورهای فرعی و زیرساخت (Async Provisioning)Manage slave servers and infrastructure (Async Provisioning)

سطح دسترسی: ادمین اصلیAccess Level: Main Admin Only

مدیریت نودها فقط با کلید ادمین اصلی امکان‌پذیر است. نمایندگان (Sub-Admins) دسترسی به این بخش ندارند. Node management is restricted to the Main Admin Key. Sub-Admins do not have access to infrastructure settings.

GET /api/v1/nodes
Main Admin Only

دریافت لیست تمام نودها، وضعیت اتصال، متد ارتباطی و حجم مصرفی کل. ترافیک از کش حافظه (`NODE_TRAFFIC_STATE`) خوانده می‌شود. Retrieve list of nodes, connection status, communication method, and total traffic usage. Traffic is read from memory cache (`NODE_TRAFFIC_STATE`).

POST /api/v1/nodes
Main Admin Only | Async (202 Accepted)

افزودن نود جدید. این درخواست بلافاصله پاسخ 202 Accepted می‌دهد و فرآیند نصب سنگین (نصب OpenVPN، تنظیم WireGuard/Direct، انتقال فایل‌ها) در پس‌زمینه شروع می‌شود. Add a new node. Returns 202 Accepted immediately. The heavy provisioning process (Install OpenVPN, Setup WireGuard/Direct, Transfer files) runs in a background thread.

پارامترهای ورودیInput Parameters

name
String
Required
نام نمایشی نود. Node display name.
ip_address
String
Required
آی‌پی پابلیک یا دامنه نود. Node public IP or domain.
ssh_password
String
Required
رمز عبور روت (SSH Root Password). Root password (SSH Root Password).
communication_method
String
Default: 'wireguard'
روش اتصال: wireguard (تونل امن) یا direct_tcp (بدون تانل). Connection type: wireguard (Secure Tunnel) or direct_tcp (No Tunnel).
sync_all_users
Boolean
Default: false
آیا تمام کاربران موجود بلافاصله به این نود اضافه شوند؟ Should all existing users be synced to this node immediately?
show_main_config
Boolean
Default: true
نمایش کانفیگ مستقیم این نود در صفحه اشتراک. Show this node's direct config on the subscription page.

PUT /api/v1/nodes/<id>
Main Admin Only | Config + SSH

ویرایش کانفیگ نود. سیستم به طور خودکار از طریق SSH به نود متصل شده، فایل server.conf را آپدیت کرده و سرویس OpenVPN را ریستارت می‌کند. از مکانیزم قفل (Node Lock) برای جلوگیری از تداخل استفاده می‌شود. Update node config. System automatically connects via SSH, updates server.conf, and restarts OpenVPN service. Uses Node Locking to prevent race conditions.

openvpn_port
String/Int
تغییر پورت اتصال. Change connection port.
protocol
String
udp یا tcp. (سیستم به طور خودکار تنظیمات explicit-exit-notify را مدیریت می‌کند). udp or tcp. (System automatically manages explicit-exit-notify settings).
ssh_password
String
آپدیت رمز عبور روت (اختیاری). Update root password (optional).

DELETE /api/v1/nodes/<id>
Main Admin Only | Async (202)

حذف نود (Master Deletion Task) Node Deletion (Master Deletion Task)

نود را برای حذف "پرچم‌گذاری" می‌کند و پاسخ فوری می‌دهد. عملیات اصلی شامل: پاکسازی دیتابیس کاربران، حذف Peer از WireGuard سرور اصلی و اجرای اسکریپت Uninstall روی سرور نود (SSH) در پس‌زمینه انجام می‌شود. Flags node for deletion and returns immediately. Background task handles: DB cleanup, WireGuard peer removal, and remote Uninstall script execution via SSH.

پاسخ نمونه Response Example 202 Accepted 
{
  "success": true,
  "message": "Node deletion accepted. Background process started."
}

POST /api/v1/public/ports
Main Admin Only

ایجاد یک پورت ورودی جدید و اختصاص آن به یک یا چند سرور. سرویس‌های OpenVPN به صورت خودکار روی سرورهای مقصد ایجاد و اجرا می‌شوند. Create a new inbound port and assign it to one or more servers. OpenVPN services are automatically provisioned and started on target servers.

POST /api/v1/sub_admins
Main Admin Only

ایجاد نماینده جدید. تعیین محدودیت حجمی (`total_usage_quota_gb`) و محدودیت سرور (`allowed_servers`) در اینجا انجام می‌شود. Create a new sub-admin. Define volume quota (`total_usage_quota_gb`) and server restrictions (`allowed_servers`) here.

username
String
Required
نام کاربری یکتا. Unique username.
password
String

max_users_limit
Integer

PUT /api/v1/sub_admins/<id>
Main Admin Only

ویرایش اطلاعات. تغییر وضعیت is_active به false تمام کاربران نماینده را غیرفعال می‌کند. فعال کردن مجدد، کاربران واجد شرایط را فعال می‌کند. Update details. Setting is_active to false disables ALL sub-users. Re-enabling reactivates eligible users.

POST /api/v1/sub_admins/<id>/reset_usage
Main Admin Only

صفر کردن حجم مصرفی کل نماینده. اگر نماینده به دلیل اتمام سهمیه (Quota Exceeded) محدود شده باشد، این دستور محدودیت را برداشته و کاربرانش را فعال می‌کند. Reset total usage counter. If Sub-Admin was restricted due to "Quota Exceeded", this action clears the flag and **re-enables** their users.

DELETE /api/v1/sub_admins/<id>
Main Admin Only

حذف عمیق Deep Delete

حذف نماینده و تمام کاربران زیرمجموعه او از دیتابیس و تمام سرورها. این عملیات بسیار حساس و غیرقابل بازگشت است. Deletes the Sub-Admin and ALL associated users from DB and all servers. Extremely sensitive and irreversible.

GET /api/v1/public/all_selectable_servers
Main Admin Only

دریافت لیست مسطح (Flat List) تمام سرورهای قابل انتخاب (شامل کانفیگ مستقیم سرور مادر، نودها و تمام تانل‌ها).
این متد برای **ساخت منوهای انتخاب سرور** در ربات‌های تلگرام (هنگام ساخت نماینده یا کاربر) بسیار کاربردی است. شناسه‌های بازگشتی (ID) دقیقاً همان‌هایی هستند که باید در فیلد allowed_servers متد ایجاد نماینده استفاده شوند. Retrieve a flat list of all selectable servers (Main direct config, Nodes, and all Tunnels).
This endpoint is essential for building **Server Selection Menus** in Telegram bots (e.g., when creating a sub-admin). The returned IDs are exactly what should be used in the allowed_servers field.
